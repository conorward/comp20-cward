Abstract of Findings

I found 3 security issues with Jialu's 2048 application:

  1. Rate Limiting
  2. SQL Injections
  3. Cross-Origin Resource-Sharing

Issues Found

  1. Rate Limiting, as I can make as many GET requests on her server as I like. Granted, there isn't any sensitive data currently stored in the database, however this vulnerability can still be a major issue with databases holding more sensitive data. Without a rate limit, hackers can try an unlimited number of credentials, thus enhancing their chances of submitting a recognized set of credentials and gaining access to the server's data.
  2. SQL Injections, as I was able to create and read data stored in the back-end database using curl. SQL Injections allow hackers to perform many malicious acts, including data tampering, acquisition of all data in a given database, and even data destruction.
  3. Cross-Origin Resource-Sharing

Conclusion

In conclusion, there are several problematic security vulnerabilities present in Jialu's application. The first, its lack of rate limiting, would be much more worrisome if this was an app that stored sensitive personal data, since rate limiting is a primary defense against private account hacking.
